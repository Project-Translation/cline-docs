# Cline 自定义指令库

本仓库旨在创建一个协作空间，让开发者可以分享、改进和利用有效的 Cline 自定义指令。通过创建并贡献到这个库中，我们可以增强 Cline 的能力，帮助开发者应对日益复杂的软件开发挑战。

## 什么是 Cline 自定义指令？

Cline 的自定义指令是一组您定义的指导方针或规则，用于针对特定任务或项目定制 AI 的行为和输出。可以将它们视为 Cline 的专门“编程”，使您能够：

-   **强制执行编码实践：** 确保代码风格一致、遵循设计模式以及特定语言或框架的最佳实践。
-   **标准化文件结构：** 规定文件命名约定、文件夹组织和项目结构。
-   **指导测试流程：** 定义生成单元测试、集成测试以及确保足够代码覆盖率的规则。
-   **自动化重复任务：** 创建指令来处理常见或繁琐的开发工作流程，提高效率。
-   **提升代码质量：** 为代码可读性、可维护性和性能优化设定标准。

通过为 Cline 提供精心设计的指令，您可以显著提高其在辅助软件开发过程中的准确性、可靠性和整体效果。

## 贡献自定义指令

我们鼓励所有技能水平的开发者将他们的自定义指令贡献到这个库中。您的贡献将为整个 Cline 社区构建一个宝贵的资源！

**提交自定义指令时，请遵循以下模板：**

### 1. 目的和功能

-   **这套指令的目标是什么？**

    -   提供一个清晰简洁的说明，阐述指令集的目标和预期用例。
    -   示例：“这套指令指导 Cline 为现有的 JavaScript 函数生成单元测试。”

-   **这套指令最适合哪些类型的项目或任务？**
    -   概述这套指令最适用的特定项目类型、编程语言或开发场景。
    -   示例：“这非常适合使用 Jest 测试框架的 JavaScript 项目。”

### 2. 使用指南（可选）

-   **使用这套指令是否有特定的步骤或前提条件？**
    -   如果您的指令需要超出在 Cline 提示中引用文件的特定步骤，请提供详细指南。
    -   示例：
        -   “在使用这套指令之前，请在项目根目录中创建一个 `tests` 文件夹。”
        -   “确保已安装 Jest 测试库。”

### 3. 作者与贡献者

-   **这套指令是谁创建的？**
    -   提供您的姓名或 GitHub 用户名以便正确署名。
-   **是否有其他人参与贡献？**
    -   感谢任何帮助改进或增强指令的合作者或贡献者。

### 4. 自定义指令

-   **提供完整的自定义指令集。**

**通过使用此模板并贡献您的自定义指令，您将帮助构建一个繁荣的 Cline 生态系统，使其成为对所有技能水平开发者来说更加多才多艺且高效的工具。**
